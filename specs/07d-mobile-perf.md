# SPEC 07d — Phase D: SP最適化・アクセシビリティ・性能検証・最終回帰

状態: **Draft** ／ 親: [SPEC-07](07-scroll-guide.md) ／ 前提: 07c Verified ／ 外部素材: 不要

## 1. 目的

新構成をSP・省モーション環境・低速端末で成立させ、全テスト＋実機で品質を確定する仕上げフェーズ。

## 2. 要件

| ID | 要件 |
|---|---|
| D-01 | SPレイアウト: 各章の3D主役を縦画面の**上1/3安全域**に、テキストパネルを下2/3に再配分（720px以下） |
| D-02 | SP軽量化: ページブルームOFF（07a既定の確認）・章02の岩12→8・章05リボン頂点半減・BRIEFINGホログラムを静止画的表現に縮退 |

| D-04 | キーボード/スクリーンリーダー: 全章テキストがDOM順で論理的に読め、CTAと早見表がTab到達可能。3D canvasは `aria-hidden` |

| D-07 | ドキュメント同期: README（サイト構成・遊び方の導線説明）・DESIGN.md（§4〜6を新構成に全面改訂）・QUALITY_PLAN/specsの状態表を更新 |
| D-08 | 実機確認手順書: ユーザーがスマホで確認すべき項目リスト（章送り・早見表・CTA・回転ガイド共存）を本SPECに付す |

## 3. 技術設計

- SP再配分（D-01）: 章の3D「ステージ位置」をIS_TOUCHで y+2.2/z-1 オフセット。テキストパネルは
  `bottom:0` sticky + 高さ60vh、背景に読解用の下グラデーション
- reduced-motion（D-03）: `REDUCED` フラグで CHAPTERS.update を「tl>=0.5で完成形を表示」の
  2値スナップに切替（実装は各章updateの冒頭で `if(REDUCED) tl = tl < .5 ? 0 : 1;`）
- パフォーマンス縮退（D-05）: 07aの軽量判定を2段化。判定はページ専用（ゲームのQualityと独立）
- CI的ガード: 全テスト連続実行スクリプト `tests/run-all.js`（phase1〜4+A〜C を順に実行し
  `window.__ALLRESULTS` に集約）を追加し、以後の変更時の回帰を1コマンド化
