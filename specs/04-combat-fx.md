# SPEC 04 — 破壊・戦闘エフェクト（Phase 2）

状態: **Draft** ／ 外部素材: **不要**（全て生成。SPEC-02ブルーム前提で効果最大化）

## 1. 目的

`tracer()` / `boom()` を中心とした戦闘フィードバックを三層化し、
「撃った・当たった・壊した」の手応えを引き上げる。SPEC-01の音と同期して演出する。

## 2. 要件

| ID | 要件 |
|---|---|
| FX-01 | 爆発を三層構成にする: ①閃光(短命PointLight+スプライト) ②衝撃波リング(拡大+フェード) ③岩片(ポリゴン破片5〜8個が回転しつつ飛散、重力なし・減速あり) |
| FX-02 | 岩片は破壊した小惑星と同系色・同質感(SPEC-03マテリアル共有)とする |
| FX-03 | 機関砲発射時にマズルフラッシュ(機体側2箇所、1〜2フレームの発光スプライト)を表示する |
| FX-04 | トレーサーを現行の線から「太さのあるグロー弾道」(円柱 or 加算スプライト連結)に強化する |
| FX-05 | 着弾時(命中フレーム)にスパーク粒(8〜12個、高速短命)を追加する |
| FX-06 | 被弾時、画面端に火花スパーク＋コックピット計器が0.3秒グリッチ表示(ノイズ走査線)になる |
| FX-07 | 爆発の瞬間、ブルーム strength を+0.3だけ80msブーストする(SPEC-02連携。未導入環境では無視) |
| FX-08 | 全エフェクトは**オブジェクトプール**で管理し、ゲーム中の new/dispose を発生させない |
| FX-09 | 同時爆発上限4。超過時は最古を即完了させる |
| FX-10 | SPでは破片数・スパーク数を50%に自動削減する |

## 3. 技術設計

### 3.1 プール設計（FX-08、`build()` で確保 / `clearFx()` 置換）

```js
const fxPool = {
  rings:   [],   // 衝撃波: RingGeometry + AdditiveBasic ×4
  flashes: [],   // 閃光スプライト + PointLight(intensity減衰) ×4
  shards:  [],   // 破片: 変形四面体メッシュ ×32 (8個×4爆発)
  sparks:  null, // スパーク: 単一Points(128頂点)を位置/寿命属性で使い回し
  muzzles: []    // マズルフラッシュスプライト ×2(左右砲門固定)
};
```

- 各要素は `visible=false` で待機。`spawnExplosion(pos, r, mat)` が一式を起動
- 更新は `loop()` 内の単一 `updateFx(dt)` に集約（現行の pulses 配列更新を置換統合）

### 3.2 各層パラメータ

| 層 | 初期値 | 寿命 | 動き |
|---|---|---|---|
| 閃光 | scale r×3 / light intensity 40 | 0.12s | intensity指数減衰 |
| 衝撃波 | r×0.8 → r×5 | 0.5s | scaleイージング(out-cubic)+opacity .8→0 |
| 破片 | 各 r×0.15〜0.3 | 0.9s | 初速 r×(6〜14)/s 放射状+個別回転3軸、速度×0.96/frame |
| スパーク | 点 | 0.25s | 初速 20〜40/s、Additive白→シアン |
| マズル | scale 1.2 | 0.07s | 2フレームで消滅、砲門位置に追従 |

### 3.3 トレーサー強化（FX-04、`tracer()` 置換）

- 現行: `Line` 2本 → 変更: `CylinderGeometry(0.06, 0.06, len, 5)` を弾道方向に向け、
  AdditiveBlending cyan、0.1sで opacity 1→0 + 半径収縮
- プール2本(左右)を使い回し。`lookAt` + quaternion で向き設定

### 3.4 計器グリッチ（FX-06、`drawDash()` 連携）

- `hit()` で `glitchUntil = now + 300` を設定
- `drawDash()` 描画末尾で `now < glitchUntil` の間: ランダム水平帯(2〜4本)を
  `globalCompositeOperation='source-over'` のノイズ矩形で上書き＋全体を1〜3px横シフト
- CRITICAL時(シールド0)はグリッチ帯を赤系にする(SPEC-05非常灯と整合)

### 3.5 音との同期（SPEC-01連携）

| タイミング | 演出 | 音 |
|---|---|---|
| fire() | マズル+トレーサー | gun() |
| 命中 | 三層爆発+スパーク+ブルームブースト | explosion(r) |
| hit() | 画面端スパーク+計器グリッチ | impact(critical) |

## 4. 受け入れ基準（AC）

| AC | 内容 | 対応要件 |
|---|---|---|
| AC-01 | 破壊時に閃光→衝撃波→破片の3層が1回の爆発として知覚できる(スロー録画で3層確認) | FX-01 |
| AC-02 | 破片の色・質感が破壊した岩と同系統である | FX-02 |
| AC-03 | 発射のたび砲口が光る | FX-03 |
| AC-04 | トレーサーが「太い光条」に見える(Before/After比較) | FX-04 |
| AC-05 | 被弾時に計器が一瞬乱れ、0.3秒で復帰する | FX-06 |
| AC-06 | 10秒間の連射・連続破壊でGC起因のスパイクなし(Performance計測) | FX-08,09 |
| AC-07 | SP相当で破片数が半減しフレーム予算内 | FX-10 |

## 5. リスク・備考

- AfterimagePassが強い残像を引くため、閃光の寿命は0.12s以下に抑える(残像で実質2倍に見える)
- 衝撃波リングはカメラ正対(billboard)にすると安っぽくなる場合がある → 爆心の接平面ランダム傾けで検証
