# SPEC 03 — 小惑星の質感・個体差（Phase 2）

状態: **Draft** ／ 外部素材: **不要**（頂点ノイズ＋Canvas生成テクスチャ。任意でPBR岩テクスチャに差替可 — SPEC-00 §4）

## 1. 目的

「全個体同形・単色グレーの12面体」を、個体差のある岩塊の群れに引き上げ、
回避対象としての実在感・視認性を高める。

## 2. 要件

| ID | 要件 |
|---|---|
| AST-01 | 各小惑星は頂点変位により**個体ごとに異なる形状**を持つ（46個が同形に見えない） |
| AST-02 | 形状には大きな起伏＋クレーター状の凹みの2スケールを含む |
| AST-03 | 共有のプロシージャル岩テクスチャ（明暗斑・ラフネス感）を適用する |
| AST-04 | 色バリエーション3系統（灰/茶灰/青灰）をランダム割当てする |
| AST-05 | 各個体は固有の軸・速度でゆっくり自転する（回転速度は視認の妨げにならない範囲） |
| AST-06 | 衝突判定は従来どおり球近似（`userData.r`）を維持し、ゲーム難度を変えない(CON-03) |
| AST-07 | ジオメトリ生成はゲーム初回起動時のみ行い、リスポーンでは再生成しない（GCスパイク防止） |
| AST-08 | SPでは変位ジオメトリの分割数を1段下げ、テクスチャを512→256pxにする |

## 3. 技術設計

### 3.1 形状生成（AST-01/02、`build()` 内）

- ベース: `IcosahedronGeometry(1, PC:3 / SP:2)`（現行Dodecahedronから変更。分割数が扱いやすい）
- 個体数ぶんのバリエーションは**8種の変形済みジオメトリをプール**し、46個に割当て
  （full個体別生成はメモリ過剰。8種×回転・スケール差で十分な見かけの多様性）
- 変位アルゴリズム（実装は関数 `deformRock(geo, seed)`）:
  1. 擬似乱数（seed付きLCG）で再現性を確保
  2. 大起伏: 頂点ごとに `p *= 1 + 0.28*n3(p*1.5+seed)`（n3=値ノイズ3D、コード内実装 約30行）
  3. クレーター: ランダム方向 k=4〜7個について `dot(dir,normal)>0.82` の頂点を最大12%凹ませる
  4. `geo.computeVertexNormals()`（flatShading廃止し滑らか+エッジ強調はノイズに任せる）

### 3.2 テクスチャ（AST-03、Canvas生成）

- 512×512（SP 256）Canvasに: ベースグレー → 値ノイズ4オクターブで明暗斑 →
  ランダム楕円の暗スポット(クレーター錯覚) → 微細ノイズ粒
- `THREE.CanvasTexture`、`colorSpace = SRGBColorSpace`
- MeshStandardMaterial: `map` + `roughness:.95` + `bumpMap`（同テクスチャ流用, `bumpScale:.06`）
- 3色系統(AST-04)は `material.color` の乗算（マテリアル3個を共有、per-mesh cloneしない）

### 3.3 自転（AST-05、`loop()` 内）

- 生成時に `userData.spin = new THREE.Vector3(rand±.15, rand±.2, rand±.15)`（rad/s）
- ループで `m.rotation.x += spin.x*dt` 等。SPでは遠方個体(z<-600)の自転更新をスキップ

### 3.4 変更しないもの（CON-03の担保）

- 個体数46 / 出現域(FX,FY,LN) / `userData.r` による当たり半径 / respawn ロジック
- 機関砲での破壊可否・スコア

## 4. 受け入れ基準（AC）

| AC | 内容 | 対応要件 |
|---|---|---|
| AC-01 | 画面内の任意の5個体を比較し、形状が明確に異なる | AST-01 |
| AC-02 | 近距離の個体にクレーター状の凹みが視認できる | AST-02 |
| AC-03 | 3色系統が混在して見える | AST-04 |
| AC-04 | 自転が確認でき、かつ回避判断の妨げにならない | AST-05 |
| AC-05 | 被弾テスト(従来の自動テスト手順)で当たり判定挙動が従来同等 | AST-06 |
| AC-06 | リスポーン発生時にフレームスパイクがない(Performanceタブで確認) | AST-07 |
| AC-07 | 性能予算内(PC60fps/SP相当<16ms) | AST-08 |

## 5. リスク・備考

- 滑らかシェーディング化で「ローポリの様式美」が失われる懸念 → 実装時にflatShading版と
  スクリーンショット比較し、ユーザー承認を得てから確定する（見た目の好みが分かれる点）
- bumpMapで質感が不足する場合の任意アップグレード: ambientCGのRock PBR(CC0)を
  512pxに縮小して同梱（+300KB〜、SPEC-00 §4） — Normal/Color 2枚構成
