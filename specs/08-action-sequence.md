# SPEC 08 — 連続アクションシークエンス化 ＋ SIGNAL TUNER撤去

状態: **Implemented**（2026-07-20 E1〜E3完了・最終run-all 145テスト ALL GREEN。実機レビュー待ち） ／ 親: [SPEC-07](07-scroll-guide.md)系の改訂 ／ 外部素材: 不要

## 1. 目的・コンセプト転換

ユーザー指示による2つの転換を行う。

1. **SIGNAL TUNER（交信ゲーム）を撤去**し、サイトとゲームを ASTEROID RUN 一本に集約する
2. 章ごとの独立ジオラマ（現行07b方式）をやめ、**「1機の機体による一本の出撃飛行」**に再構成する。
   スクロールは飛行の進行であり、説明は**アクションの決定的瞬間で時が止まる（フリーズビート）**
   ことで挿入される — 例:「岩に激突する瞬間で画面が止まり、シールドの説明が始まる」

> **新しい物語フレーム**: 閲覧者は出撃した1機の僚機に随伴している。
> スクロールするほど機体は小惑星帯を進み、事件（回避・砲撃・被弾）が起きるたびに
> 時間が凍結して教官の説明が入る。凍結が解けると結末（爆発の完了・シールド喪失）が再生され、
> 飛行は続く。最後に帯を抜け、「次はお前が飛ぶ番だ」— LAUNCH DECK でゲーム起動。

## 4. 技術設計

### 4.1 タイムワープ（ACT-03 — フリーズの数理）

```js
// 平坦区間 [f0,f1] を持つ時間写像。フリーズ中 ta は一定値 f0 を返す
// tl: 章ローカル進行度 / ta: アクション進行度(演出はすべて ta の純関数)
function warp(tl, f0, f1){
  if(tl <= f0) return tl / f0 * f0;              // 前半: 等速(そのまま)
  if(tl <= f1) return f0;                        // フリーズ: 停止
  return f0 + (tl - f1)/(1 - f1)*(1 - f0);       // 後半: 残りを再生
}
const freeze = (tl, f0, f1)=>tl > f0 && tl < f1; // フリーズ判定(テキスト点灯とHUDに使用)
```

- 演出関数は従来どおり純関数のまま、引数が `tl` から `ta=warp(tl,…)` に変わるだけ
- テキスト行の`data-on`閾値はフリーズ区間内のtlに割当（凍結中に行が順に点灯）
- 解除音(ACT-05)は f1 の上向きエッジ検出（既存B-06方式）

### 4.2 連続機体（ACT-01 — Sortieレイヤー）

- `Guide` を改修し、章別ステージとは別に **Sortieレイヤー**（機体1機・共通岩帯・排気）を新設
- 機体のワールド位置はグローバルtの純関数: `shipPath(t)` = カメラパスの少し前方を蛇行
  （章のイベント座標=激突点・砲撃点はshipPath上の固定tに配置）
- 章ステージは「その章のイベント道具」（標的岩・シールド面・爆発・惑星）だけを持つ
- カメラはCAMキーフレームを増補し、フリーズ中は対象へ寄る（例: 激突点を斜め上から見るアングル）

### 4.3 フリーズ演出（ACT-04）

- 減彩: フリーズ深度 `fz∈[0,1]`（区間端でイーズ）で全画面CSSフィルタ
  `filter: saturate(1-fz*.5) contrast(1+fz*.08)` を`#space`へ — DOM側で完結し3D負荷ゼロ
- スピードライン/塵: 速度係数に `(1-fz)` を乗算（凍結で静止）
- HUD: `TIME HOLD //` バッジをフリーズ中のみ表示（既存hud-status隣）
- SEはフリーズ進入時に低いdrone一拍（`ui('hold')`を新設: 220Hz sine 300ms）

### 4.4 SIGNAL TUNER撤去手順（REM）

1. `SignalGame` IIFE・`wave-canvas`・関連CSS・`btn-contact`/`deck-contact`・SIGNAL早見表カードを削除
2. `SoundEngine.beatOsc` 削除。`openLayer('signal')` 分岐・ESC/回転ガイド等の共通基盤はasteroid専用に単純化（分岐は残しても害はないため、削除は参照箇所のみ）
3. テスト改訂: phase1(AUD-10系1件削除)・phase4(SIG系4件削除→PRF/BG系のみに)・phaseB(COMM章4件削除)・phaseC(実通信/CTA系修正)・run-all更新。**ASTEROID RUN系は全件維持**
4. 文書: README(遊び方をASTEROID RUN単独に)・DESIGN(§5-2撤去注記)・specs 00表更新
