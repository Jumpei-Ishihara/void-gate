## 2. テスト戦略

- warp写像は**純関数のユニットテスト**が可能（期待値表6点）— フリーズ機構の正しさを最初に固定
- 「凍結中は動かない」= FZ内2点で全イベント道具の座標一致、という強い検証で保証
- 音・凍結エッジは既存B-06のエッジ検出テストパターンを流用
- 撤去(E1)は**grep相当の存在チェック**をテスト化（AC-01の識別子リスト）
- 各Phase完了時に run-all（E1で改訂）を全実行。ゲーム本体テストは全期間無変更・全グリーン維持

## 3. リスク管理

| リスク | 対策 |
|---|---|
| 凍結中に読み切れない/読後に間延び | FZ幅・章高さを定数テーブル化し、実機レビュー後に数値のみで再調整 |
| 機体とイベント道具の位置ズレ（レイアウト変化時） | イベントtをmeasure()毎に再計算し従属配置（07b実証済みの方式） |
| 撤去の削除漏れ | 08a §2の識別子リストをテスト化して機械検証 |
| E2中の本番劣化 | E1完了時点は「現行構成-SIGNAL」の完成状態でデプロイ可能。E2はSortie追加→旧章置換の順で、置換完了までデプロイしない |
| 凍結の視覚合図が弱い | 減彩+バッジ+音の3チャンネル。レビューで不足ならビネット追加（定数1つ） |
