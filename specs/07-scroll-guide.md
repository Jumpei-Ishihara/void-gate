# SPEC 07 — スクロール3D展開型「遊び方ガイド」サイト再構成

状態: **Draft**（要件整理＋デザイン検討。実装は承認後） ／ 外部素材: **不要**（既存3D資産・SoundEngineを全面流用）

## 1. 目的・コンセプト

サイト全体を**2つのミニゲームの「遊び方の説明」**として再構成する。
ただし読み物のマニュアルにはせず、**スクロールに応じて3D宇宙空間が展開していく
「出撃前ブリーフィング」体験**として設計し、説明そのものが世界観の演出になるようにする。

> **物語フレーム**: 閲覧者は深宇宙ステーション「VOID GATE」に着任した新任オペレーター。
> ページを下るごとに艦の発進シークエンスが進行し、
> 訓練（＝遊び方の説明）を修了した者だけが実戦（＝ゲーム起動）へ向かう。
> 「スクロール＝発進準備の進行」「読了＝出撃資格の取得」という意味づけを与える。

- 説明の各ステップは**実物の3Dオブジェクト**（本物の小惑星・コア・機体・波形）で示す — 図解ではなく実物
- 既存の世界観資産（ネオンHUD・Orbitron・グリッチ・ブルーム調の発光素材・SoundEngine）を継承
- 最終セクションが**LAUNCH DECK（出撃甲板）**となり、2つのゲームCTAに合流する

## 3. デザイン検討

### 3.1 章構成（7章 + 発進シークエンス）

| # | 章 | 伝えること | 3D展開（スクロール連動） |
|---|---|---|---|
| 00 | **GATE** (HERO) | タイトル・着任通知・「下へ＝発進準備」 | 現行ヒーロー。ゲートリング越しに深宇宙。スクロールキューを「発進シークエンス開始 ▼」に変更 |
| 01 | **BRIEFING** | 2つの任務の全体像（探査/交信） | カメラが前進、左右に2つのホログラム（岩塊 / 波形リング）が浮かび、任務章へ分岐する予告 |
| 02 | **FLIGHT** | 操縦: マウス/ドラッグで機体追従、上下左右 | **小惑星帯へ進入**。実物の変形岩（deformRock流用）が視差で流れ、中央に機体（ゲーム実装と同モデル）が現れてカーソル/指に僅かに追従するライブデモ |

| 05 | **COMM** | SIGNAL TUNER: 横=周波数/縦=振幅、5ch同調 | 空間に**3D波形リボン**が展開し、スクロールでマゼンタ波形へ吸い付いて同調→同心円パルス。末尾に交信フォーム（MIG-02） |
| 06 | **NAVIGATION** | （世界観章）航行宙域の選択＝惑星セレクタ | 現行PLANETSを再文脈化。チップ選択で3D惑星が変わる体験を「操作が世界に届く」練習として提示 |
| 07 | **LAUNCH DECK** | 出撃: 2ゲームのCTA・ベストスコア・操作早見表 | カメラがゲートを抜け、視界が開けて2基の発進口（EXPLORE/CONTACT）。操作早見表はHUDパネル意匠で常設表示 |

### 3.2 スクロール脚本（タイムライン設計）

全長を仮に 800vh とし、進行度 t∈[0,1] を章ローカル進行度に分配する。代表章の脚本例:

```
02 FLIGHT (t: .18〜.34)
  .18-.22  岩帯進入: 岩12個が遠方から視差流入、章タイトル グリッチイン
  .22-.30  機体がフェードイン→カーソル/指追従デモ(±小可動)。テキスト行1「マウスで操縦」点灯
  .30-.34  テキスト行2「上下も機敏に動く」→機体がピッチ演出→章退出(岩が後方へ)

03 WEAPONS (t: .34〜.50)
  .34-.40  コア結晶が奥から正面へ(スケール+リング回転はt駆動)。行1「◆コアを回収」
  .40-.44  行2「左クリック/FIREで機関砲」→ t=.44 でトレーサー2条(位置はtの関数)
  .44-.48  t=.44-.48 で三層爆発の各層をt割当(閃光→リング→破片)。行3「コア1つで岩を砕く」
  .48-.50  「空撃ちは消費なし / NO CORE警告」注記 → 退出
```

- **全演出はtの純関数**（SCR-05）: `explosionProgress = clamp((t-.44)/.04)` のように実装し、
  逆スクロールでは爆発が逆再生される（これ自体がSF的で世界観に合う）
- 章タイトル・本文はDOM（現行reveal機構を拡張し、章内の行単位表示もt駆動に変更）

### 3.3 スクロール駆動方式の比較検討

| 方式 | 概要 | 長所 | 短所 | 判定 |
|---|---|---|---|---|
| A. カメラフライスルー | 3D空間に全章の実体を配置し、カメラがZ航行で通過 | 「三次元に展開」の実感が最大。現行のZ前進を自然拡張 | 章間の距離設計が難しい。全実体が同時にシーンに存在 | ◎ **採用（基幹）** |
| B. 章内アセンブリ | カメラ固定気味で、章ごとにオブジェクトが組み上がる | 各章の演出自由度が高い | 空間の連続感が弱い | ○ 章内表現として併用 |
| C. スクロールジャック | ホイールを乗っ取り章単位スナップ | 演出制御が容易 | 操作感が悪くアクセシビリティに難。世界観と不一致 | ✕ 不採用 |

**採用: A+Bハイブリッド** — カメラは常にZへ航行し（A）、各章の主役はカメラ前方の
「ステージ位置」で t 駆動の組み上がり/退出を行う（B）。
ピン留めはDOM側を `position: sticky` で行い、3Dはカメラ相対配置で追従する。

### 3.4 ビジュアル言語（世界観の担保）

- **色**: 章ごとにアクセントを割当 — FLIGHT=シアン / WEAPONS=白熱色（爆発） /
  SURVIVAL=マゼンタ→赤 / COMM=マゼンタ×シアン干渉 / LAUNCH=フルパレット。基調は現行の深宇宙#030014
- **タイポ**: 章番号 `BRIEFING 02 / 07 — FLIGHT` をOrbitronレタースペース広めで統一。
  重要操作語（クリック・ドラッグ・FIRE）はHUDチップ意匠（`.ghud-row`流用）でインライン強調
- **音**(VIS-05): 章進入で `ui('select')` の薄いトーン、WEAPONS爆発で `explosion()`、
  SURVIVAL で `impact(true)`、LAUNCH DECK でエンジンドローン微起動。
  AudioContext解禁前（初回クリック前）は無音のまま演出が成立すること
- **HUD**: 既存の四隅フレーム・セクター表示を `BRIEFING n/7` 表記に更新（GDE-06）

### 3.5 SP・アクセシビリティ

- SPは縦スクロールで同一脚本。3D主役の配置を縦画面の安全域（上1/3=3D、下2/3=テキスト）に再配分
- 操作説明はIS_TOUCHで文言・図像を出し分け（GDE-04）。SPにキーボード表記を出さない
- reduced-motion: カメラ航行を静止＋クロスフェードに縮退（SCR-06）。逆再生演出も無効
- スクロールが不能な環境（キーボード閲覧）: 全テキストはDOMとして常在し、Tab到達可能（GDE-07）

## 4. 技術設計（概要 — 実装SPECは承認後に詳細化）

- **タイムライン**: `const T = scrollY / (docHeight - innerHeight)` をRAF内でサンプリング（PER-04）、
  `smoothT += (T - smoothT) * dt*4` の慣性つきで全駆動系に配布
- **章定義テーブル**: `CHAPTERS = [{id, t0, t1, enter(tl), update(tl), exit(tl)}]` 形式で宣言的に管理
- **3D資産の流用**: `deformRock`/コア構築/機体構築/波形描画は現在AsteroidRun/SignalGame内部にあるため、
  **共有ファクトリ（`buildRock(seed)`, `buildCore()`, `buildShip()`）としてモジュールスコープへ抽出**し、
  ゲームとページの両方から参照する（挙動不変のリファクタ。既存テストで回帰確認）
- **ページ側ブルーム**(VIS-04): 既存UnrealBloomPassをページcomposerにも適用（PC のみ、strength低め0.5）。
  ゲーム起動中はページ側レンダラ停止（現行仕様）のため二重負荷はない
- **実装フェーズ分割案**:
  - Phase A: タイムライン基盤+章テーブル+カメラパス（既存セクションのまま駆動系だけ差替え）
  - Phase B: 章コンテンツ実装（02〜05の3D主役と脚本）
  - Phase C: LAUNCH DECK・NAVIGATION 再文脈化・HUD/コピー全面差替え
  - Phase D: SP最適化・reduced-motion・性能検証・回帰

## 6. リスク・論点（承認前に判断が必要な点）

| # | 論点 | 推奨 |
|---|---|---|
| 1 | ページ全長（現行5セクション≒500vh → 8章≒800vh）。長すぎると離脱リスク | 章あたり80〜120vhで**総長700〜800vh**に収め、HUDに進行度を常時表示して先が見える設計にする |
| 2 | NAVIGATION（惑星）章の要否 — 純粋な遊び方説明には不要 | 世界観と「操作が世界に届く」学習価値のため**存置を推奨**（MIG-01）。削る場合は7章→6章に短縮可 |
| 3 | ページ側ブルームの負荷（VIS-04） | PCのみ・低strengthで採用推奨。予算超過ならページ側のみ即OFF（ゲームに影響なし） |
| 4 | 共有ファクトリ抽出はゲームコードのリファクタを伴う | 既存テスト70項目が回帰網として機能するためリスク低。Phase Aの前に単独で実施し回帰を先に取る |
| 5 | 交信フォームの位置（COMM章末尾 vs LAUNCH後のフッター） | 「訓練→実通信」の物語順でCOMM章末尾を推奨（MIG-02） |
