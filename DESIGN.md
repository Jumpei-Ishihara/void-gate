# VOID GATE — 設計書

宇宙 × 近未来をモチーフにした、インタラクティブWebサイトのデザイン仕様。

## 1. コンセプト

「スクロール＝ワープ航行、カーソル＝観測装置」。
閲覧者を"宇宙船のオペレーター"に見立て、ページ操作のすべてが宇宙空間への干渉として
リアルタイム3D描画に反映される体験型サイト。

- **トーン**: 深宇宙のダーク基調 + ネオンHUD（近未来SF）
- **技術**: Three.js (WebGL) によるフル3D背景 + DOMオーバーレイUI
- **言語**: 日本語コピー + 英語タイポグラフィ（Orbitron）の混植

## 5. ミニゲーム（HEROのCTAから起動）

ボタンの意味に沿って、ゲーム性の異なる2種類を設計。どちらも全画面レイヤー（z-index:50）で起動し、
ESCまたはEXITボタンでサイトに復帰する。スコアは`localStorage`にベスト記録を保存。

### 5-1. EXPLORE // 探査開始 → **ASTEROID RUN**（反射神経・回避型）

スターウォーズのアステロイドベルト追撃シーンをモチーフにした3D飛行ゲーム。

- **操作**: マウス/タッチで機体を上下左右に操縦（バンク・ピッチの傾き表現付き）。**左クリックで機関砲**（SPは左下FIREボタン）。
  サウンドは **Mキー / HUDのSOUNDボタン** でいつでもミュート切替（設定は`localStorage`保持）
- **目的**: 小惑星46個のフィールドを回避飛行しつつ、エネルギーコア◆を回収
- **機関砲**: コアを1つ消費して正面の小惑星を破壊できる（初期コア2個）。
  正面の照準コーン内で最も手前の小惑星を自動ロックし、ロック中はレティクルが増光。
  発射時は砲口のマズルフラッシュ＋機体両舷からの**円柱グロートレーサー**、
  命中時は**三層爆発**（閃光PointLight＋拡大する衝撃波リング＋破壊した岩と同質感の岩片飛散）
  ＋着弾スパーク＋反動シェイク＋ブルーム一時ブースト。
  目標がいない空撃ちはコアを消費せず、コア切れ時は計器とHUDに「NO CORE」警告（マゼンタ）。
  連射間隔160ms。UIボタン上のクリックでは発射しない。
  コアは「温存すればスコア（×500）/ 使えば道が開く」のリソース管理要素を兼ねる
- **難度曲線**: 速度が時間とともに110→480（表示440→1920 km/s）へ上昇（加速感はスピードライン粒子で演出）。
  高速域でも小型小惑星をすり抜けないよう、衝突判定はフレーム移動区間内の最接近点で行う（トンネリング対策）
- **リスク**: シールド3枚が3発まで吸収し、シールドなしの4発目で機体大破（**最大被弾4回**）。
  シールド0時はHUD・計器とも「CRITICAL」警告（マゼンタ）を表示し、
  コックピットの**赤色非常灯が1.2秒周期で明滅**する（`prefers-reduced-motion`時は常灯）。
  被弾時はマゼンタのダメージビネット＋計器の0.3秒ノイズグリッチ＋衝突点スパーク＋1.6秒の無敵点滅、
  さらに**速度が20%低下**（下限110）し通常の加速度で再加速する
- **スコア**: 航行距離×10 ＋ 残コア×500。ゲームオーバー画面でRETRY/EXIT
- **フィールドの質感**: 小惑星は頂点変位（大起伏＋クレーター凹み）した**8種のジオメトリプール**
  ×3色系統（灰/茶灰/青灰）×Canvas生成の岩テクスチャ（bumpMap兼用）で個体差を持ち、
  各個体が固有の3軸でゆっくり自転する。当たり判定は従来の球近似のまま（難度不変）。
  エネルギーコアは発光八面体＋**軌道傾斜つき二重リング（異速回転）**の結晶体
- **空間演出**: 進行方向の遠景にバイオレットの**リング付き大型惑星**（フォグ対象外）、
  スピードラインより遅い層速度で流れる**塵レイヤー600粒**（視差でスピード感を強調）
- **サウンド**: 全SFXをWeb Audio合成（音声ファイル0個） — 機関砲/爆発（岩の大きさで重さが変化）/
  被弾（CRITICAL時は警告ブザー付き）/コア取得アルペジオ/UI音、および**速度連動ピッチの
  エンジンドローン**。同時発音8ボイス上限・指数ランプ減衰でクリックノイズを防止。
  ゲーム終了時に全音停止（詳細は specs/01-audio.md）
- **実装**: Three.js別シーン（専用WebGLRenderer、メインの背景レンダラはゲーム中は描画停止して負荷を回避）。
  戦闘エフェクトは全て**オブジェクトプール制**（ゲーム中のnew/dispose無し、同時爆発上限4）

#### 視点モード（COCKPIT / CHASE）

起動メニューで選択でき、ゲーム中も **Vキー / HUDボタン** で即時切替可能。

- **COCKPIT（操縦席視点・デフォルト）**: カメラにコックピット一式を子付けした一人称視点
  - **操縦桿**: 入力に連動してリアルタイムに傾く（ドーム型ベース＋カプセルグリップ＋シアン発光リング＋マゼンタボタン）
  - **スロットルレバー**: 現在速度に連動して角度が変わる（円筒ロッド＋横置きカプセルノブ＋発光リング）
  - **計器ディスプレイ**: 独立した3面パネル（各面に専用の `CanvasTexture` を毎4フレーム描画）を
    ラップアラウンド配置 — 左右パネルはヨー±0.42radで操縦者側へ内向きに角度づけ、中央はやや奥・下に配置して
    操縦席の立体感と包まれ感を演出。内容は**速度計（270°アークメーター）**/ **近接レーダー**（実際の小惑星・コアの
    相対位置をプロット、近距離はマゼンタ警告色、スイープライン付き）/ システム状態（シールド残量は1枚以下で警告色、コア取得数）。
    速度計は針＋進捗アーク＋目盛り＋上位12%のレッドゾーンで最高速への近さが直感的に分かり、
    上限到達時は数値が**MAX表示**（マゼンタ発光、メーターもマゼンタ化）に変わる
  - **HUDレティクル**: 四隅のコーナーブラケット＋中央十字のスクエアデザイン。
    色・透明度は常時固定（薄いシアン / opacity 0.12）で視界を遮らない。
    **自動ロックオンモード（AUTO・デフォルト）** — 通常の約半分の狭い正面コーン内で最も手前
    （＝次にぶつかる）小惑星に吸い付き、対象の見かけサイズに合わせて拡縮（脈動付き）。
    ロックは位置と拡縮のみで表現し、「囲んでいるもの＝撃てば壊れるもの」を示す。
    機関砲も同じ狭いコーンで発射されるため照準表示と砲撃結果が常に一致する。
    **通常モード（OFF）** — レティクルは画面中央より下の定位置で操舵方向に追従する飾りに戻り、
    機関砲は従来の広いコーンで発射。**Lキー / HUDボタン「LOCK: AUTO/OFF」でいつでも切替可能**
  - **視界の確保**: 飛行中はカーソルの追従リングを非表示にし、ドットのみ5pxの半透明シアンに減光して残す
    （カーソルがウィンドウ外に出た際の「操作不能と誤認」を防ぐ位置インジケータとして機能）。
    視界上部を横切るフレーム類は置かず、視界は計器・レティクル・ビネットのみで構成
  - **造形**: 角を排した曲面構成 — 操縦者を包み込む円筒アークの曲面コンソール＋発光トリム。
    コンソール下部/側部に**グリーブル16個**（材質別に結合しdraw call抑制）、計器3面に立体ベゼル、
    非同期周期で明滅する小型インジケータランプ3灯を配置。コックピット内シアン照明、
    CSSによるキャノピービネット。CRITICAL時は赤色非常灯が点灯（リスク欄参照）
- **CHASE（追跡視点）**: 機体後方からの三人称視点。機体はエンジンノズル×2＋**速度連動で伸びる
  排気グロー**＋翼端灯（左:赤/右:緑）を備え、被弾時は無敵点滅。
  コックピット視点では機体を非表示にして写り込みを防止

#### スムージング / スピード演出

- **慣性カメラ**: 視線・カメラロール・カメラピッチ・操縦桿・レティクルはすべて指数補間（イージング）で追従し、カクつきを排除
- **上下機動**: 可動域は左右46に対し上下34、追従レートは上下dt×8（左右dt×6より機敏）。
  上下機動時はロールと同じ慣性構造の**機首上げ/下げ（カメラピッチ、±0.4radクランプ）**が入り、
  バンクと対称のリアリティを持たせている
- **ポストプロセス**: `EffectComposer` で **Render → UnrealBloomPass（発光） → AfterimagePass（残像）** の直列合成。
  ブルームでネオン・コア・トレーサー・計器が物理的に「にじんで光り」、
  残像は減衰係数が速度に連動して0.68→0.88まで上昇（速いほど世界が流れる）。
  コックピットはカメラ固定のため常にシャープで、外界だけがブラーする。SPはブルーム半解像度
- **スピードライン**: 280本の `LineSegments` 流線。線の長さが速度に比例して伸びる
- **FOVキック**: 速度に応じて視野角72°→84°へ滑らかに拡大（加速感の強調）
- **衝突インパクト**: 被弾時は二重の衝撃演出 — 3Dカメラの高周波ランダム振動（位置＋ロール、減衰付き）と、
  画面全体（HUD含む）のCSSキーフレームによる0.45秒の小刻み振動、ダメージビネットを同時に再生

### 5-2. CONTACT // 交信 → **SIGNAL TUNER**（精密操作・同調型）

「交信」の意味に合わせた周波数チューニングゲーム。オシロスコープ風2D Canvas描画。

- **操作**: マウスX=周波数 / マウスY=振幅。自機波形（シアン実線）をターゲット波形（マゼンタ破線）に重ねる
- **ルール**: 同調を1.3秒維持でチャンネル確立（SYNCバー表示）。許容誤差はチャンネルごとに15%ずつ縮小
- **勝利条件**: 制限時間60秒以内に5チャンネル同調 → 断片メッセージが**1文字ずつランダム文字から
  確定していくデコード演出**で全文出現する物語演出
- **画面演出**: 波形は太い下引き＋コアラインの二重描画グロー。チャンネル確立の瞬間は
  **同心円パルス＋擬似フラッシュ**（`prefers-reduced-motion`時は省略）。
  背景に事前生成のCRT走査線パターン＋周辺減光を重ね、判読性を保ちながらブラウン管質感を付与
- **フィードバック**: ズレに比例して自機波形にノイズジッタ。共有サウンドエンジン管理の
  2オシレータによる「うなり」で耳でも同調度がわかる（ミュートはHUDボタンで切替・全ゲーム共通）
- **スコア**: チャンネル×1000 ＋ 残り時間×20

## 6. ページ構成（スクロール3D展開型「出撃前ブリーフィング」8章制 — SPEC-07）

サイト全体が遊び方の説明を兼ねる。スクロール進行度を単一タイムライン(`Tl`)が管理し、
キーフレームカメラのZ航行と各章の3Dステージ（章ローカル進行度tlの**純関数**＝逆スクロールで逆再生）を駆動する。

| # | 章 | 内容 / 3D展開 |
|---|---|---|
| 00 | GATE | 着任通知タイピング + 「▼ 発進シークエンス開始」 |
| 01 | BRIEFING | 2任務のワイヤーフレームホログラム（岩塊/波形リング） |
| 02 | FLIGHT | 岩帯進入 + 実物機体のカーソル/指追従ライブデモ |
| 03 | WEAPONS | コア結晶接近 → トレーサー → 三層爆発（tl駆動） |
| 04 | SURVIVAL | シールド3枚の段階破壊 + CRITICAL赤ビネット |
| 05 | COMM | 3D波形リボンの同調収束 + パルス。直後に実通信フォーム |
| 06 | NAVIGATION | 惑星セレクタ（操作が世界に届く最終訓練） |
| 07 | LAUNCH DECK | 発進口CTA×2 + ベストスコア + 操作早見表（CONTROLS単一ソース） |

- 章のt範囲はDOM実測から自動導出（レイアウト変更に追従）。テキスト行は章内tl閾値で点灯
- HUDは章表記（GATE 00/07〜）と連動。`prefers-reduced-motion`時は各章が静的完成形+全行即時点灯に縮退
- 詳細設計: specs/07〜07d

## 7. スマートフォン(SP)対応

### サイト本体（≤720px）
- ナビゲーションと右端プログレスバーを非表示、セクション余白・ゲームパネル余白を縮小
- カード・惑星チップ・フォームは1カラムに自然折返し（flex-wrap）
- `@media(hover:none)` でカスタムカーソルを非表示にしネイティブ操作へフォールバック

### ゲームは横画面でプレイ（SP）
- ミニゲームはSPでは**横画面専用**。コックピット計器3面が視界に収まり、左右の操舵幅も確保できるため
- **フルスクリーンはモード選択時にオプトイン（デフォルトON）**: 両ゲームの起動メニューに
  「フルスクリーンでプレイ」チェックボックスを配置（選択は`localStorage`に記憶）。
  ONのままLAUNCH/TUNE INすると、Fullscreen API（webkitプレフィックス対応）で全画面化して
  ブラウザのツールバー等の干渉を排除し、ゲーム終了時に解除。SPはさらに
  `screen.orientation.lock('landscape')`で横画面へ自動固定（Android Chrome等）。
  PCではESCがフルスクリーン解除＝ゲーム終了と一致する。
  API非対応環境（iPhone Safari等）は通常表示のまま、縦持ちの間
  **「ROTATE DEVICE / 横画面でプレイしてください」ガイド**（回転するスマホアイコン付き）を全画面表示
- 縦持ちガイド表示中は**ゲームを完全一時停止**（時間・距離・被弾も進まず、SIGNAL TUNERは音も停止）。
  横に回転すると即座に再開する。ガイド表示中もEXITボタンは操作可能
- ゲーム終了時はフルスクリーンと画面固定を解除して通常のページ閲覧に戻る
- **モーダルの収まり**: 横持ちなど低い画面（高さ≤560px）ではメニュー/リザルトのモーダルを
  圧縮表示（余白・文字サイズ縮小）して1画面に収める。それでも収まらない場合は
  **モーダル自身が内部スクロール**する（タッチ対象＝スクローラーとなるためiOSでも確実に動作。
  `margin:auto`＋`max-height`で通常時は中央寄せ、はみ出し時も上端が切れない）
- **ゲーム中HUDの圧縮（タッチ端末）**: ステータスを短縮ラベルの小型チップ
  （DIST / CORE / シールド残量アイコン）で**上端1行**に横並び表示し、視界の占有を最小化。
  キーボード前提のVIEW(V)/LOCK(L)ボタンはタッチ端末では表示しない
  （視点は起動メニューで選択、ロックオンはAUTO固定）。警告色（NO CORE・CRITICAL）は維持
- **FIREボタンは左下**に配置（右手で操舵・左手親指で射撃の分担）

### ゲームのタッチ操作
- **操舵**: `touchstart` / `touchmove` で指を置いた瞬間から追従（両ゲーム共通）
- **機関砲（ASTEROID RUN）**: タッチ端末では右下サムゾーンに**円形FIREボタン**を表示
  （`hover:none` または `ontouchstart` で判定し、ゲーム開始時のみ出現）。
  タップで単発、**長押しで連射**（発射間隔は`fire()`内の160msクールダウンに従う）。
  `preventDefault`で合成マウスイベントを抑止し、画面タップ由来の合成mousedownは
  直近800msのタッチ記録で無視（誤射・機体ワープ防止）。
  起動メニューの操作説明もタッチ端末では「右下のFIREボタン」表記に切替
- **HUDのコンパクト化（≤720px）**: ステータスパネルをEXITボタンと重ならない幅
  （left:12px / right:104px）に収め、フォント・パディングを縮小して縦の占有を圧縮。
  VIEW / LOCK 切替はキーボードなしでもHUDボタンで操作可能
- **SIGNAL TUNER**: SYNCバーを狭幅時はコンパクトHUDの直下（y=176）に移動し重なりを解消。
  バー幅も画面幅に応じて縮小（最大280px）
- コックピット計器の左右パネルは縦持ちでは画面外に切れるが、
  速度・シールド・コア残量はDOM側HUDで常時確認できるため情報欠落はない

## 8. パフォーマンス / アクセシビリティ配慮

- `devicePixelRatio` を最大2にクランプ、パーティクルは `depthWrite:false` で描画負荷を抑制
- ページ側のパルスは寿命管理し `geometry/material.dispose()` でGPUメモリを解放。
  ゲーム側の戦闘エフェクトは**オブジェクトプール制**でゲーム中の生成/破棄を排除
- **性能自動調整（adaptive quality）**: ゲーム中のフレーム時間を移動平均で監視し、
  予算超過（PC>12ms / SP>20ms）が3秒続くと品質を1段階縮退
  （①ブルーム半解像度 → ②破片/塵50% → ③ブルームOFF）。同一セッション内では戻さない（振動防止）。
  SPは既定でブルーム半解像度始まり。段階はコンソールログで確認可能
- `prefers-reduced-motion` でリビール・スクロールキュー・同調パルス・非常灯明滅を無効化/常灯化
- タッチデバイスではカスタムカーソルを非表示にしネイティブ操作へフォールバック
- スクロールリスナーは `passive:true`
