# VOID GATE — 素材クオリティ向上 方針書

> **SDD詳細設計**: 本方針の実装仕様は [specs/](specs/00-overview.md)（SPEC 00〜06）を参照。
> 実装はSPECごとの承認(Approved)後に着手する。

ゲーム体験の質を左右する素材（音・光・造形・エフェクト）を、現行の制約を守ったまま
引き上げるための検討結果と実施計画。

## 1. 前提となる制約（方針の土台）

| 制約 | 内容 | 方針への影響 |
|---|---|---|
| 完全オフライン | 外部CDN・外部アセット禁止（配布物 約1.7MB） | 画像・音声ファイルの追加はせず**プロシージャル生成**を基本とする |
| 単一HTML構成 | index.html にCSS/JS内包（現行 約1,800行） | 素材はコードで記述。肥大化するようならJS分割を検討 |
| SP対応 | 横画面プレイ・ミドルレンジ端末想定 | 品質はデスクトップ基準で作り、**SPは自動で軽量化**する二段構え |

**基本方針: 「アセットファイルを増やさず、生成ロジックの質で勝負する」。**
テクスチャはCanvas生成、音はWeb Audio合成、造形は頂点加工。追加容量ほぼゼロで
オフライン性を維持しながら、見た目と手触りを引き上げる。

## 2. 現状の棚卸しとギャップ

| 素材 | 現状 | ギャップ（質を下げている点） |
|---|---|---|
| 音響 | **ASTEROID RUNは無音**。SIGNALのみ2オシレータ | 射撃・爆発・被弾・取得・警告すべて無音。体感の半分を損失 |
| 発光表現 | ブルームなし（残像のみ） | ネオン/HUDが「光って見えない」。世界観の要が未達 |
| 小惑星 | 単一12面体・単色グレー・フラットシェード | 全個体同形。岩の質感・個体差・自転がない |
| 爆発 | 点パーティクル飛散のみ | 破片・衝撃波・閃光・残光がなく手応えが軽い |
| 機体/コア | プリミティブ組合せ・単色 | CHASE視点で見栄えが平板。コアは八面体+スプライトのみ |
| コックピット | 曲面コンソール+計器3面（実装済みで良好） | 細部ディテール（配管・ボルト・スイッチ・非常灯）が無く「型」感 |
| 背景宇宙 | 星3層+スプライト星雲（実装済みで良好） | ゲーム中の奥行き演出（遠景惑星・塵）が薄い |
| SIGNAL画面 | 2D線描画+グリッド | 発光の重み・CRT質感・同調成立の祝祭感が控えめ |

## 3. 改善方針（素材カテゴリ別）

### A. サウンドデザイン【最優先・費用対効果最大】
Web Audio APIによる**全SFXプロシージャル合成**（音声ファイル0個）。
- 機関砲: ノイズバースト+矩形波の減衰（連射に耐える軽量ボイス管理）
- 爆発: ローパスノイズ+サブベースの衝撃、距離減衰
- 被弾: 金属的インパクト+警告ブザー、シールド喪失で音程変化
- コア取得: 上昇アルペジオ（報酬感）
- 環境: エンジンドローン（速度連動ピッチ）、コックピット内ハム
- UI: ホバー/決定/カウントダウン
- ミュートボタンをHUDに常設、初回操作でAudioContext解禁（自動再生制限対応）

### B. ブルーム(発光)ポストプロセス【世界観の要】
`UnrealBloomPass` をローカル同梱（依存シェーダ含め+3〜4ファイル・数十KB）。
- ネオン・コア・トレーサー・爆発・計器が「本当に光る」
- 残像パスと直列合成: Render → Bloom → Afterimage
- SPはブルーム解像度を1/2にして負荷を抑制（品質チェックボックスと連動も検討）

### C. 小惑星の質感・個体差
- 頂点ノイズ変形で**1ジオメトリから個体ごとに異なる岩塊**を生成（クレーター凹み含む）
- Canvas生成のプロシージャルテクスチャ（明暗斑+AO風陰影）を共有適用
- 色相バリエーション（灰/茶/青灰）+ サイズ別ディテール段階
- ゆっくり自転させ「漂う岩」の実在感を出す

### D. 破壊・戦闘エフェクト
- 爆発を三層化: **閃光(PointLight)+衝撃波リング+ポリゴン破片**（岩片が回転しながら飛散）
- 機関砲: マズルフラッシュ、トレーサーの太さ/グロー強化、着弾スパーク
- 被弾: 画面端スパーク+コックピット計器の一瞬のグリッチ表示

### E. 機体・コア・コックピットの造形
- 機体(CHASE用): 翼・エンジンノズル・排気グロー（速度連動で伸縮）を追加
- コア: 二重回転リング+内部発光の「エネルギー結晶」化（ブルームと相乗）
- コックピット: ボルト/配管/小型ランプ等のグリーブル、CRITICAL時に赤色非常灯が明滅

### F. 背景・SIGNAL TUNER
- ゲーム背景に遠景惑星と塵レイヤーを追加し速度感と奥行きを強化
- SIGNAL: 波形グロー強化・同調成立時のパルス演出・微CRT歪み（ブルーム流用）

## 4. パフォーマンスガードレール

- パーティクル/破片は**プール制**で生成・破棄を抑制（GC/スパイク防止）
- SP自動軽量化: ブルーム半解像度・破片数50%・テクスチャ512px・星の数削減
- フレーム時間を監視し、閾値超過で段階的にエフェクトを自動縮退（adaptive quality）
- 目標: デスクトップ60fps / ミドルSP 40fps以上

## 5. 実施計画（フェーズ分け・推奨順）

| Phase | 内容 | 体感インパクト | 規模 | 状態 |
|---|---|---|---|---|
| **1** | A サウンド一式 + B ブルーム | ★★★★★（別物になる） | 中 | ✅ Verified (テスト17/17) |
| **2** | C 小惑星の質感 + D 破壊エフェクト | ★★★★ | 中 | ✅ Verified (テスト20/20) |
| **3** | E 機体/コア/コックピット造形 | ★★★ | 中 | ✅ Verified (テスト14/14) |
| **4** | F 背景/SIGNAL + 4 の性能自動調整仕上げ | ★★★ | 小〜中 | ✅ Verified (テスト19/19+全回帰) |

**2026-07-20 全Phase実装完了**。テスト駆動（tests/phase1〜4.js 計70項目）で検証済み。

Phase 1 を最優先とする理由: 「無音→音がある」「発光しない→光る」は
既存素材すべての見え方・手触りを一括で底上げする横断的改善であり、
個別素材を作り込む Phase 2 以降の効果もブルーム前提で最大化されるため。
